CodeForge ships with opinionated defaults, but nearly every part of the environment is configurable.

Use this section when you want to:

- change model or permission defaults
- enable or disable plugins and components
- customize system prompts and rules
- configure auth and secrets
- add hooks or custom keybindings

## What This Section Covers

- runtime behavior in `settings.json`
- container behavior in `devcontainer.json`
- prompt, rule, and hook customization
- secret and token management
- optional and disabled-by-default components
