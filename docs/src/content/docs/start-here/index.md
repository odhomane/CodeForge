This section is the fastest path from zero to a working CodeForge session.

If you are new to the project, follow these pages in order:

1. [Before You Install](./before-you-install/) — check prerequisites, supported clients, and hardware expectations.
2. [Install in VS Code](./install-in-vscode/) — the recommended setup path.
3. [Verify Your Install](./verify-install/) — run the built-in health checks.
4. [Start Your First Session](./first-session/) — launch `cc` and learn the default session model.
5. [Your First Tasks](./your-first-tasks/) — try real prompts and workflows.

If you are not using VS Code, use one of the alternate setup pages instead of the recommended install path:

- [Use the DevContainer CLI](./devcontainer-cli/)
- [Other Clients](./other-clients/)

If you are upgrading an older install, use [Migrate to v2](./migrate-to-v2/).

## What this section does

- Keeps the first-run path linear
- Puts alternate clients behind a separate branch
- Defers deep configuration and internals until after you have a working session

## After onboarding

- Go to [Use CodeForge](/use/) for day-to-day workflows
- Go to [Customize](/customize/) when you want to change defaults
- Go to [Reference](/reference/) when you need exact command or config details
